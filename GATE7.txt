SECTION I: THE MANDATORY PERSISTENCE TEST
===================================================================

This is the single test with real teeth. If this fails, DeNovoAI cannot
deliver on its core promise. Do not ship until this section passes entirely.

  1. Log in as test user
  2. Navigate to /studio → start a new chat session
  3. Describe an app: "A marketplace for freelance dog walkers"
  4. Continue through clarifying questions until stage reaches "confirming"
  5. Verify /studio/confirm/[sessionId] loads with correct app details
  6. Verify slot_map data matches what was described (app name, template)
  7. Click "Deploy it" — verify app record is created in /dashboard
  8. Navigate to /dashboard — verify the new app appears in the list
  9. Navigate away (go to /pricing), then back to /dashboard
  10. Verify the app is STILL in the list
  11. Hard refresh /dashboard (Ctrl+R) — verify app still present
  12. Click the app card → verify /dashboard/app/[appId] loads with correct details
  13. Log out → log back in → verify the app is still in My Apps list

  [ ] PASS — app record created and survives navigation
  [ ] PASS — app record survives hard refresh
  [ ] PASS — app record survives logout + re-login

  IF ANY FAILS: DeNovoAI cannot retain generated apps. Data is lost.
  Fix Supabase RLS / user_id assignment before continuing.
